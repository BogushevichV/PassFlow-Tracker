# PassFlow Tracker

Система анализа пассажиропотока городского транспорта.

## Стек технологий

| Компонент | Технология |
|-----------|-----------|
| Язык | C# (.NET 8) |
| UI-фреймворк | Avalonia UI 11 |
| База данных | PostgreSQL 16 (Docker-контейнер) |
| Драйвер БД | Npgsql |
| Архитектура | MVVM (CommunityToolkit.Mvvm) |
| Связь с БД | IPC через TCP (127.0.0.1:5000) |

---

## Системные требования

- **Windows** 10/11 (64-bit) или **Ubuntu** 20.04/22.04/24.04 (64-bit)
- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Windows) или Docker Engine (Linux)
- ОЗУ: не менее 4 ГБ
- Диск: не менее 2 ГБ (включая образ PostgreSQL ~300 МБ)
- Интернет: требуется при первом запуске для загрузки образа PostgreSQL

---

## Установка

### Windows

1. Установите [Docker Desktop](https://www.docker.com/products/docker-desktop/) и запустите его
2. Распакуйте архив `PassFlowTracker-v1.0-win-x64.zip`
3. Откройте PowerShell **от имени администратора** и выполните:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\install.ps1
```

Установщик скопирует файлы в `C:\Program Files\PassFlow Tracker\`, создаст ярлыки на рабочем столе и в меню Пуск.

**Удаление:** Пуск → PassFlow Tracker → «Удалить PassFlow Tracker»

### Linux (Ubuntu/Debian)

```bash
sudo apt update && sudo apt install docker.io
sudo dpkg -i passflow-tracker_1.0.deb
```

---

## Запуск

### Windows
Дважды щёлкните ярлык **PassFlow Tracker** на рабочем столе.

### Linux
```bash
passflow-tracker
```

При первом запуске приложение автоматически скачает образ PostgreSQL и создаст контейнер `my_postgres`. Это занимает 1–5 минут.

## Использование

### Импорт данных

1. Нажмите **[Загрузить JSON]** в разделе «ФАЙЛ» на ленте инструментов
2. Выберите файл `.json` с данными пассажиропотока
3. Дождитесь завершения импорта (статус в строке состояния)

### Аналитические функции

#### Часы пик
Определяет час суток с максимальным пассажиропотоком и строит гистограмму по всем 24 часам.

1. Нажмите **[Часы пик]** в разделе «АНАЛИЗ»
2. Выберите область: **Вся сеть** или **Конкретный маршрут**
3. Нажмите **[Показать]** — откроется вкладка «ГРАФИКИ» с гистограммой

#### Топ остановок
Находит N остановок с наибольшим пассажиропотоком.

1. Введите количество остановок в поле под кнопкой (по умолчанию 10)
2. Нажмите **[Топ ост.]**
3. Выберите режим агрегации:
   - **По отдельности** — каждая запись отдельно с датой и временем
   - **Сумма за день** — суммирование по остановке за каждый день
   - **За всё время** — суммирование за весь период

#### Низкая активность
Находит рейсы, где вошедших пассажиров меньше заданного порога.

1. Введите пороговое значение в поле под кнопкой (по умолчанию 10)
2. Нажмите **[Низк. акт.]** — результат отобразится во вкладке «Рейсы»

### Вкладки данных

| Вкладка | Описание |
|---------|----------|
| **Все данные** | Иерархическое дерево: День → Круг → Рейс → Остановка |
| **Дни** | Сводные данные по каждому дню |
| **Круги** | Данные по каждому кругу маршрута |
| **Рейсы** | Данные по каждому рейсу |
| **Остановки** | Данные по каждой остановке |

### Экспорт данных

Перейдите на нужную вкладку и нажмите в разделе «ФАЙЛ»:
- **[JSON ↓]** — экспорт в формат JSON
- **[Excel ↓]** — экспорт в формат .xlsx (файл откроется автоматически)

## Настройки подключения

Файл `appsettings.json` в папке установки:

```json
{
  "Docker": {
    "ImageName": "postgres",
    "Tag": "latest",
    "ContainerName": "my_postgres",
    "Password": "mysecretpassword",
    "DbPort": "5532"
  },
  "ConnectionStrings": {
    "MainConnection": "Host=localhost;Port=5532;Username=postgres;Password=mysecretpassword;Database=passflowtrackerdb"
  },
  "Ipc": {
    "Host": "127.0.0.1",
    "Port": 5000,
    "AuthToken": "my-secret-token"
  }
}
```

## Устранение неполадок

**«Docker daemon недоступен»**
Запустите Docker Desktop и дождитесь иконки кита в системном трее.

**Приложение долго запускается при первом запуске**
Идёт загрузка образа PostgreSQL (~300 МБ). Последующие запуски будут быстрее.

**«Ошибка подключения к базе данных»**
Проверьте что Docker Desktop запущен и контейнер работает:
```bash
docker ps
docker start my_postgres  # если контейнер остановлен
```

**Гистограмма не отображается**
Убедитесь что данные импортированы, затем нажмите кнопку «Часы пик» заново.
